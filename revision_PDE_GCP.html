Rôle principal

Data warehouse serverless et entièrement managé. Optimisé pour l'analyse OLAP (pas OLTP). Facturation à la donnée scannée. Supporte des pétaoctets de données avec des requêtes SQL standards.

Concepts clés (~30% de l'examen)

Partitionnement Clustering STRUCT / ARRAY Vues matérialisées BigQuery ML Slots Authorized Views BI Engine

Partitionnement vs Clustering

PartitionnementClustering
Divise physiquement la table par colonne (date, entier)Trie les données par colonnes fréquemment filtrées
Réduit les données scannées → réduit le coûtRéduit les données lues → améliore la vitesse
Max 1 colonne de partitionJusqu'à 4 colonnes de clustering
Ex: partition par date de commandeEx: cluster par pays, catégorie produit

Exemple SQL

CREATE TABLE dataset.commandes PARTITION BY DATE(date_commande) CLUSTER BY pays, categorie AS SELECT * FROM dataset.raw;

Optimisation des requêtes

Éviter SELECT * · Filtrer sur la colonne de partition · Utiliser des vues matérialisées pour les requêtes répétitives · Analyser le plan d'exécution via INFORMATION_SCHEMA

✅ Règle d'or

Toujours partitionner par date en premier, puis clusterer par les colonnes WHERE les plus utilisées. Éviter SELECT * sur une table partitionnée — ça scanne TOUT.

⚠️ Piège fréquent

BigQuery ML = modèles simples directement en SQL. Pour des modèles plus complexes, utiliser Vertex AI. Les deux se complètent et ne s'excluent pas.

Rôle principal

Service de messagerie publish/subscribe entièrement managé. Découple les producteurs et consommateurs. Garantit la livraison des messages même si le consommateur est hors ligne.

Composantes clés

Topic Subscription Publisher Subscriber Acknowledgement Dead Letter Queue

Push vs Pull Subscription

PullPush
Le subscriber demande activement les messagesPub/Sub envoie automatiquement vers un endpoint HTTPS
Idéal pour traitement batch ou variableIdéal pour webhooks et Cloud Functions
Contrôle total sur le rythme de consommationLatence plus faible, plus simple à configurer

Flux typique dans un pipeline

Sources → Pub/Sub Topic → Dataflow (transformation) → BigQuery (analytique) → Cloud Storage (archive)

✅ Règle d'or

Pub/Sub = boîte aux lettres intelligente. Le facteur (publisher) dépose, le destinataire (subscriber) relève quand il veut. Toujours acquitter (ACK) après traitement.

⚠️ Piège fréquent

Messages non acquittés → réenvoyés automatiquement → doublons. Toujours implémenter l'ACK après traitement réussi pour éviter les retraitements.

Rôle principal

Service managé de traitement de données basé sur Apache Beam. Un seul pipeline Python/Java fonctionne en batch ET en streaming sans modifier le code. Serverless et autoscalable.

Concepts clés Apache Beam

PCollection PTransform Pipeline Windowing Watermark Triggers Templates Autoscaling

Types de fenêtres (Windowing)

TypeDescriptionUsage
FixedFenêtres de durée fixe sans chevauchementRapport toutes les 5 minutes
SlidingFenêtres glissantes avec chevauchementMoyenne mobile sur 1 heure
SessionFenêtre basée sur l'activité utilisateurSessions web analytics

Dataflow vs Dataproc

DataflowDataproc
Serverless, pas de clusterCluster Hadoop/Spark à gérer
Nouveaux pipelines from scratchMigration jobs Hadoop existants
Batch + Streaming unifiedSpark, Hive, Pig spécifiques

✅ Règle d'or

Pour tout nouveau pipeline sur GCP → Dataflow. Pour migrer des workloads Hadoop/Spark existants → Dataproc. La règle est simple et testée fréquemment.

⚠️ Piège fréquent

Watermark ≠ Trigger. Le watermark indique quand les données tardives ne seront plus acceptées. Le trigger définit quand émettre les résultats intermédiaires d'une fenêtre.

Rôle principal

Service managé pour exécuter des workloads Apache Hadoop, Spark, Hive et Pig sur GCP. Idéal pour migrer des pipelines existants vers le cloud ou des jobs Spark complexes.

Concepts clés

Cluster Jobs Spark Hive HDFS → GCS Autoscaling Cluster éphémère Preemptible VMs

Quand choisir Dataproc ?

DataprocDataflow
Migration de jobs Hadoop/Spark existantsNouveaux pipelines from scratch
Workloads Hive, Pig, HBase nécessairesPipelines batch et streaming unifiés
Contrôle fin sur la configuration clusterServerless, zéro gestion d'infrastructure
Plus économique pour jobs longs et intensifsPlus simple à opérer et maintenir

✅ Règle d'or

Si la question mentionne "migration Hadoop", "jobs Spark existants" ou "Hive" → réponse = Dataproc. Pipeline nouveau → Dataflow. Cette distinction est testée à chaque examen.

⚠️ Best Practice

Utilisez des clusters éphémères (créés pour un job, supprimés après) et stockez sur GCS plutôt que HDFS. Réduction des coûts jusqu'à 70% avec les Preemptible VMs.

Rôle principal

Service d'orchestration de workflows managé basé sur Apache Airflow. Planifie, séquence et monitore des pipelines complexes avec des dépendances entre tâches via des DAGs Python.

Concepts clés

DAG Operator Task Scheduler XComs Hooks Connections

Structure d'un DAG Python

from airflow import DAG from airflow.operators.python import PythonOperator with DAG('pipeline_pde', schedule_interval='@daily') as dag: t1 = PythonOperator( task_id='extraire', python_callable=extraire) t2 = PythonOperator( task_id='charger_bq', python_callable=charger) t1 >> t2 # Séquence

✅ Règle d'or

Composer = chef d'orchestre. Il ne traite pas les données — il dit à Dataflow, BigQuery et Cloud Storage quand et comment travailler ensemble. Pas de traitement temps réel.

⚠️ Piège fréquent

Cloud Composer n'est pas pour le temps réel. Pour le streaming → Pub/Sub + Dataflow. Composer est pour l'orchestration batch planifiée (quotidien, hebdomadaire, etc.).

Rôle principal

Stockage objet scalable et durable pour tout type de fichier. Sert de landing zone pour les données brutes avant transformation, et d'archive pour les données historiques à long terme.

Classes de stockage

ClasseUsage
StandardDonnées actives, accès fréquent
NearlineAccès ~1x/mois
ColdlineAccès ~1x/trimestre
ArchiveAccès ~1x/an, backup long terme

Concepts clés

Bucket Object Lifecycle rules Versioning Signed URLs IAM + ACL Transfer Service

✅ Règle d'or

GCS = disque dur universel de GCP. Toutes les données brutes y passent avant BigQuery. Utilisez les Lifecycle rules pour migrer automatiquement vers Coldline/Archive après X jours.

⚠️ Piège fréquent

Ne pas confondre GCS (stockage objet non structuré, fichiers) et BigQuery (stockage analytique structuré, tables SQL). GCS stocke des fichiers, BigQuery stocke des tables.

Rôle principal

Plateforme ML unifiée de GCP pour entraîner, évaluer, déployer et monitorer des modèles ML. Directement lié à la mission Anthropic — maîtriser Vertex AI est un différenciateur fort.

Composantes clés

Workbench AutoML Custom Training Feature Store Model Registry Pipelines Endpoints Model Monitoring

AutoML vs Custom Training vs BigQuery ML

AutoMLCustom TrainingBigQuery ML
Pas de code ML requisContrôle total sur l'architectureSQL uniquement
Prototypage rapideModèles complexes/spécifiquesDonnées déjà dans BigQuery
GCP gère toutCode d'entraînement customPas de mouvement de données

✅ Lien Anthropic

Vertex AI Feature Store = gestion centralisée des features pour des modèles cohérents. C'est exactement ce dont Anthropic a besoin pour ses pipelines d'évaluation de Claude.

⚠️ Data Drift — Concept clé

La dérive des données dégrade silencieusement les modèles ML. Vertex AI Model Monitoring détecte automatiquement les changements de distribution et déclenche des alertes.

Rôle principal

La sécurité des données est le différenciateur clé chez Anthropic. IAM, DLP, VPC Controls et Audit Logs forment le socle de tout pipeline de données sécurisé en production.

Concepts clés

IAM Cloud DLP VPC Controls CMEK Audit Logs Dataplex Data Catalog Authorized Views

IAM — Principe du moindre privilège

Type de rôleDescriptionUsage
PrimitifsOwner, Editor, ViewerÀ éviter en production — trop larges
PrédéfinisBigQuery Data Viewer, Dataflow Worker…Recommandés pour la plupart des cas
PersonnalisésPermissions exactes nécessairesCas très spécifiques, granularité max
Compte de serviceIdentité pour les applicationsJamais pour les humains

Flux sécurisé recommandé

Source → Cloud DLP (masquage PII) → Dataflow (transformation) → BigQuery (stockage chiffré) → Audit Logs (traçabilité)

✅ Règle Anthropic

La sécurité des données n'est pas optionnelle — c'est le cœur de la mission Anthropic. Pensez "sécurité en premier" dans chaque décision de pipeline lors de l'entretien.

⚠️ Piège fréquent

Les Data Access Logs de BigQuery NE sont PAS actifs par défaut. Ils doivent être activés explicitement dans IAM → Audit Logs. Erreur classique à l'examen et en production.
